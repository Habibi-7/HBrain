## Install

Use the installer so platform wrappers are generated from the canonical
`skill/SKILL.md`:

```bash
sh install.sh --cursor    # or --claude, --windsurf, --copilot
```

Then create your vault and copy templates:

```bash
mkdir -p ~/brain/events ~/brain/renders ~/brain/.brain/templates
cp skill/templates/* ~/brain/.brain/templates/
```

### Other agents

The skill is a self-contained markdown document. Load `skill/SKILL.md` wherever
your agent platform accepts skills, rules, or system prompt additions.

## Set a vault path

Tell your agent where to store your vault, or set `$BRAIN_DIR`:

```bash
export BRAIN_DIR=~/brain
```

If `$BRAIN_DIR` isn't set, the agent will ask you once and remember the path.

## Capture your first event

Just talk to your agent:

> "I think Postgres is the right call because of its native JSON support."

The agent creates a file like:

```
~/brain/events/2026/05/04/01JVMY7QX-decided-postgres.md
```

With contents:

```markdown
---
id: 01JVMY7QXR8KF3DNQJ5CGPXG9S
schema: 1
type: decision
created_at: 2026-05-04T14:32:11Z
source: agent
agent: claude-code
tags: [backend, database]
links: []
---

I think Postgres is the right call because of its native JSON support.
```

The agent confirms: "Captured decision · `01JVM...`"

## Query your events

> "What did I decide about the database?"

> "Show my week"

> "Show my week as a visual timeline"

> "List my open tasks"

The agent reads your vault files and responds. Simple questions can stay in
chat. Visual requests should become self-contained HTML artifacts.
