## Ask in natural language

> "What did I decide about the database?"

> "Show my tasks from last week"

> "What did I capture about Postgres?"

> "List my open tasks"

The agent translates your question into file tool calls.

## How it works under the hood

The agent uses shell commands against your vault:

```bash
# Recent events (last 7 days)
find $BRAIN_DIR/events -name "*.md" -mtime -7 | sort

# Filter by type
grep -rl "^type: decision" $BRAIN_DIR/events/ | sort

# Filter by tag
grep -rl "tags:.*backend" $BRAIN_DIR/events/ | sort

# Search by content
grep -rl "postgres" $BRAIN_DIR/events/ | sort

# Open tasks
grep -rl "^status: open" $BRAIN_DIR/events/ | sort
```

You don't run these yourself — the agent does. But knowing the mechanism helps you understand why specific phrasing finds things.

## Marking a task done

> "Mark the Postgres integration task as done."

The agent finds the file, changes `status: open` to `status: done`, and writes it back.

## Tips for findability

- **Use consistent tags.** If you tag architecture decisions `architecture`, search by that tag later.
- **Put key terms in the body.** The agent searches body text, not just frontmatter.
- **Link related events.** The `links` field connects events for richer retrieval.
