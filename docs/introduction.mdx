# Living Brain

**Markdown memory. LLM judgment. HTML artifacts.**

Your agent already knows how to reason and write files. This skill teaches it
what is worth keeping, how to structure it, and when to turn it into a rich view.

```
You speak naturally → Agent captures durable signals → Agent answers or renders HTML views
```

## How it works

Install the skill into your agent. Then talk naturally:

- "I think Postgres is the right call because of the JSON support"
- "I need to review the proposal by Friday"
- "What did I decide about the database?"
- "Show my week as a timeline"

The agent captures events as markdown files in a vault folder you own. Simple
questions get answered directly. Visual requests become self-contained HTML
artifacts. Everything durable stays plain text — sync it with Git, iCloud,
Obsidian, or whatever you already use.

## What the skill teaches

| Capability | What it does |
|-----------|-------------|
| **Schema** | How to structure events (5 types: note, task, decision, fact, link) |
| **Capture** | How to notice durable signals without relying on exact trigger phrases |
| **Query** | How to find, filter, and aggregate events using file tools |
| **Views** | When to answer in text vs. generate self-contained HTML artifacts |
| **Helpers** | How to use optional scripts/CLI for deterministic work without depending on them |

## Custom skills

The default second-brain skill is just the beginning. Create domain-specific skills for any workflow:

```bash
brain skill create "Meeting Notes" "Track meetings and action items"
brain skill create "Research Tracker" "Papers, citations, reading notes"
```

Each skill gets its own `SKILL.md` — load it into your agent to activate a completely different capture workflow with custom event types and triggers. Skills share the same plain-markdown format, so they work with any agent that reads files.

## Supported agents

Any agent that can read a markdown skill file and use file tools:

- Claude Code / Cowork
- Cursor
- Manus
- Any other agent with filesystem access
