## Layout

```
~/brain/                               (or $BRAIN_DIR)
├── events/
│   └── YYYY/MM/DD/
│       └── <ulid>-<slug>.md          # one file per event
├── renders/                           # saved view artifacts
├── skills/                            # custom skills
│   └── <skill-name>/
│       ├── SKILL.md                   # skill definition
│       └── vault/
│           └── events/               # isolated event storage
└── .brain/
    └── templates/
        └── timeline.html             # view templates
```

## events/

The source of truth. One markdown file per event, organized by date.

File naming: `<ulid>-<slug>.md`
- `<ulid>` — 26-char time-sortable unique ID
- `<slug>` — 2-5 lowercase words from the body, hyphenated, max 48 chars

Example: `01JVMY7QXR8KF3DNQJ5CGPXG9S-chose-postgres-json-support.md`

## renders/

Saved artifacts from rendered views (timelines, digests). These are derived — you can delete them and regenerate anytime.

## .brain/templates/

Strict view templates with `{{placeholder}}` slots. The agent fills the slots with event data to produce consistent views. See [Templates](/reference/templates).

## Syncing

The vault is a plain folder. Sync however you already sync files:

- **Git** — version history, branching, multi-device
- **iCloud / Dropbox / Syncthing** — transparent sync
- **Obsidian Sync** — if you browse with Obsidian

Living Brain doesn't build its own sync.

## skills/

Custom skill directories created by `brain skill create`. Each one has:

- `SKILL.md` — the skill definition. Load this into your agent to activate the skill.
- `vault/events/` — event storage isolated from the main vault.

See [Custom Skills](/guides/skills) for how to create and modify skills.

## Manual editing

Open any event file in any editor. Rules:
- **Don't change `id` or `created_at`** — immutable identifiers
- Everything else (body, tags, links, status) is fair game
