## What templates do

When you ask for a view ("show my week", "my open tasks"), the agent:

1. **Queries** — finds and reads relevant event files
2. **Loads** the matching template from `.brain/templates/`
3. **Fills** the `{{placeholder}}` slots with event data
4. **Presents** the rendered view

Templates define stable default layouts. The agent fills slots when a template
matches, but templates are not cages. If the user asks for a new design, the
agent can generate a fresh HTML artifact while following the artifact rules.

## Available templates

| File | Use when |
|------|---------|
| `timeline.html` | "Show my week", "my timeline", "recent events" |

Templates are installed to your vault with:

```bash
cp skill/templates/* ~/brain/.brain/templates/
```

## Adding templates

Drop any file with `{{placeholder}}` slots into `.brain/templates/`. The agent
will use it when a view request matches.

Teams sharing a vault can share templates — everyone sees identical views.

## Fallback

If no template exists for a visual request, the agent generates a self-contained
HTML artifact.

HTML artifact rules:

- Inline CSS and SVG; no external network dependencies.
- Semantic HTML and readable contrast.
- Include provenance: date range, filters, event count, and generation time.
- Never invent source data.
- Do not mutate the vault from generated HTML.
